Política de privacidad — Sparkmaster

Esta Política de privacidad se aplica a la aplicación móvil Sparkmaster y al sitio web sparkmaster.app.

1. Finalidad de la aplicación

Sparkmaster es un compañero de mantenimiento y servicio para motores fuera borda. La aplicación admite varias marcas de motores y ayuda a los usuarios a registrar datos del motor, historial de mantenimiento, comprobaciones, fotos y otras notas relacionadas con la navegación.

Sparkmaster está diseñada principalmente como una aplicación offline-first. Esto significa que los datos principales de la aplicación se almacenan por defecto en el propio dispositivo del usuario y no se envían a los servidores de Sublime Web Oy sin una acción o consentimiento aparte del usuario.

2. Sitio web

Sparkmaster.app es un sitio web estático. El sitio no ofrece cuentas de usuario, formularios ni funcionalidad de pago, y el propio sitio web no recopila datos personales directamente.

Si un usuario nos contacta por correo electrónico, Sublime Web Oy recibe la información que el usuario decida enviar, como la dirección de correo del remitente, el contenido del mensaje y los archivos adjuntos.

3. Información procesada por la aplicación

3.1 Datos almacenados en el dispositivo

La aplicación puede almacenar en el dispositivo del usuario los siguientes tipos de datos:

  • Datos del motor, como modelo, número de serie, año, horas de uso y nombre de la embarcación
  • Historial de mantenimiento, como trabajos realizados, fechas, piezas, pasos y notas personales
  • Fotos añadidas por el usuario, por ejemplo del motor, piezas o pasos de mantenimiento
  • Listas de comprobación y notas de servicio
  • Datos de viaje, como coordenadas GPS, duración del viaje, horas de motor y destino opcional, si el usuario inicia el seguimiento manualmente o activa el seguimiento automático del puerto
  • Historial de diagnósticos, como resultados de resolución de problemas y resúmenes de escaneo de SparkPilot que el usuario elige guardar
  • Datos de contacto opcionales del taller o servicio (por ejemplo correo electrónico), si el usuario los introduce en ajustes
  • Identificador aleatorio (UUID) si el usuario utiliza funciones de comunidad o contribución

Por defecto, estos datos se almacenan localmente en el dispositivo del usuario.

3.2 Datos de ubicación

Sparkmaster puede procesar datos de ubicación solo si el usuario utiliza el seguimiento de viaje o activa el seguimiento automático del puerto (opcional).

Seguimiento manual de viaje: El usuario inicia un viaje por sí mismo. Mientras el viaje está activo, la aplicación registra puntos GPS espaciados para calcular horas de motor y duración. Durante el viaje se muestra una notificación persistente (servicio en primer plano). La ubicación también puede leerse con la pantalla apagada, pero solo mientras el viaje esté activo. Los puntos de ruta se almacenan en el dispositivo.

Seguimiento automático del puerto (opcional): El usuario puede activar por separado un geofence alrededor de su puerto base (Viajes → Seguimiento automático del puerto). Al salir del área definida, un viaje puede iniciarse automáticamente; al regresar, puede finalizar. Esto utiliza geofencing de Android, que puede requerir permiso de ubicación en segundo plano (ACCESS_BACKGROUND_LOCATION) para funcionar de forma fiable cuando la aplicación está cerrada o la pantalla apagada. La función está desactivada por defecto.

Permisos y opciones: Los permisos de ubicación se solicitan solo después de que el usuario vea una explicación en la aplicación. La ubicación en segundo plano se solicita solo para el seguimiento automático del puerto. El usuario puede denegar el permiso o revocarlo en cualquier momento en los ajustes del dispositivo.

Los datos de ubicación no se envían por defecto a los servidores de Sublime Web Oy. Si no se usa el seguimiento de viaje ni el seguimiento automático del puerto, la aplicación no procesa datos de ubicación.

3.3 Fotos

El usuario puede añadir fotos a la aplicación, por ejemplo del motor, piezas o pasos de mantenimiento. Por defecto, las fotos se almacenan en el dispositivo del usuario.

Si el usuario comparte posteriormente una foto mediante una función de comunidad o la envía con un informe de error, es responsable de asegurarse de que la foto no contenga datos personales, matrículas, rostros, datos de ubicación u otro contenido que no desee compartir.

3.4 Funciones de comunidad y contribución

Si el usuario decide publicar una contribución comunitaria, como información de compatibilidad de piezas, experiencia con herramientas, un consejo de mantenimiento o una foto, la aplicación puede procesar y publicar el contenido que el usuario haya elegido compartir específicamente.

Esto puede incluir:

  • Identificador aleatorio (UUID)
  • Texto compartido por el usuario
  • Fotos compartidas por el usuario
  • Números de pieza, información del modelo, información de herramientas y otro contenido proporcionado voluntariamente por el usuario

Las contribuciones comunitarias son voluntarias. La aplicación no publica nada automáticamente.

El identificador aleatorio (UUID) no es el nombre ni el correo electrónico del usuario, pero puede ser un identificador técnico persistente. Por precaución, se trata como dato personal cuando puede vincularse a contenido publicado.

Las contribuciones comunitarias publicadas pueden licenciarse bajo la licencia Creative Commons Attribution 4.0 International (CC BY 4.0). Esto significa que otros pueden utilizar el contenido publicado, siempre que se atribuya la contribución original según los términos de la licencia.

Antes de publicar, el usuario debe asegurarse de que el contenido compartido no contenga datos personales u otra información que no desee hacer pública.

3.5 SparkPilot e IA en el dispositivo

Si el usuario activa SparkPilot, Sparkmaster puede procesar respuestas sobre síntomas, fotos seleccionadas, muestras cortas del sonido del motor, mediciones de RPM y un resumen minimizado de los datos del motor y mantenimiento. La inferencia de IA se ejecuta localmente en el dispositivo. Este contenido no se envía a Sublime Web Oy ni a un proveedor de IA.

Las fotos y grabaciones originales se eliminan por defecto después del análisis. Las fotos adjuntas en el chat de SparkPilot también se procesan solo en el dispositivo y se borran tras el turno o al cerrar la sesión (no se suben). El usuario puede conservar por separado una referencia del motor sano o guardar un resultado en el historial local de diagnósticos. Los mensajes del chat de IA se mantienen solo durante la sesión actual y se borran al salir del chat. Los modelos 3D esquemáticos del chat son ilustraciones propias de la app, no diagramas de piezas OEM. El modelo de IA descargable se almacena localmente y puede eliminarse en los ajustes. El Engine Health Passport solo se crea tras confirmación y excluye por defecto el número de serie, la ubicación exacta, los medios originales y las notas libres.

SparkPilot ofrece apoyo a la decisión, no un diagnóstico garantizado. Las advertencias críticas utilizan reglas fijas de la aplicación además de la IA. El usuario debe verificar las instrucciones de mantenimiento y seguridad con la documentación oficial del motor.

4. Información que no recopilamos en nuestros servidores

Por defecto, Sparkmaster no almacena lo siguiente en los propios servidores de Sublime Web Oy:

  • Cuentas de usuario o contraseñas
  • Nombre del usuario
  • Historial continuo de ubicación en nuestros servidores
  • Identificadores publicitarios para redes de anuncios
  • Perfiles en redes sociales
  • Análisis de comportamiento con fines de marketing o publicidad
  • Registros completos de mantenimiento, fotos o rutas de viaje (salvo que el usuario los envíe voluntariamente por correo electrónico)

La aplicación puede almacenar localmente en el dispositivo un correo electrónico opcional del taller si el usuario lo introduce. El formulario de informe de errores no solicita ni almacena el correo del usuario en la aplicación; si el usuario envía correo desde su propia aplicación de email, su dirección puede ser visible para el destinatario para poder responder.

Si el usuario envía un informe de error o una solicitud de soporte por correo electrónico, el contenido del mensaje y la dirección del remitente quedan visibles para Sublime Web Oy.

5. Servicios de terceros

Sparkmaster no utiliza redes publicitarias de terceros ni análisis de comportamiento con fines de marketing.

La aplicación puede utilizar los siguientes servicios de terceros:

Google Play (compras dentro de la aplicación)

Las funciones premium pueden adquirirse a través de Google Play. Google procesa los pagos y los registros de compra. Sublime Web Oy no recibe ni almacena datos de tarjetas de pago o bancarios.

Política de pagos de Google: https://policies.google.com/payments

Firebase Crashlytics (Google)

En compilaciones de producción con Firebase configurado, la aplicación puede enviar diagnósticos de fallos y errores a Google Firebase Crashlytics para mejorar la estabilidad. Esto puede incluir, por ejemplo:

  • Trazas de pila de fallos y mensajes de error
  • Versión y número de compilación de la aplicación
  • Modelo del dispositivo y versión del sistema operativo
  • Contexto técnico no personal, como la marca o familia del motor seleccionado (si está disponible)

Crashlytics no se utiliza para recopilar notas de mantenimiento, fotos, rutas de viaje o números de serie de sus registros. Puede desactivar los informes de fallos en los ajustes de la aplicación (Ajustes → Compartir datos de uso anónimos). Las compilaciones de depuración desactivan siempre la recopilación.

Firebase Analytics (Google)

En compilaciones de producción con Firebase configurado, la aplicación puede enviar análisis agregados del producto a Google Firebase Analytics para comprender el uso de las funciones y mejorar la aplicación. Esto no se utiliza para publicidad ni perfiles de marketing.

Ejemplos de lo que puede enviarse:

  • Nombres de eventos personalizados cuando confirma voluntariamente información de piezas o comparte una cifra de consumo de combustible con el flujo comunitario
  • Parámetros no personales como marca del motor, familia del modelo, categoría de pieza, nivel de confianza, unidad de combustible o valores numéricos de consumo que haya elegido compartir
  • Versión de la aplicación y contexto técnico similar a Crashlytics

Analytics no recopila su nombre, correo electrónico, número de serie, notas de mantenimiento, fotos, rutas de viaje ni ubicación exacta. Puede desactivar Analytics en los ajustes de la aplicación (Ajustes → Compartir datos de uso anónimos). Las compilaciones de depuración desactivan siempre la recopilación.

Información de privacidad de Firebase / Google: https://firebase.google.com/support/privacy

Otro uso de red

La aplicación puede abrir enlaces en el navegador del usuario (por ejemplo páginas de distribuidores) y descargar fuentes u otros recursos necesarios para la interfaz. Esos servicios no se utilizan para crear perfiles publicitarios.

Si añadimos más servicios (por ejemplo sincronización en la nube o cuentas de usuario), esta Política de privacidad se actualizará antes o cuando se introduzcan.

6. Cómo se utilizan los datos

Los datos almacenados en la aplicación se utilizan principalmente para permitir al usuario gestionar su propio motor, mantenimiento e información de navegación.

Sublime Web Oy utiliza las solicitudes de soporte e informes de error recibidos por correo electrónico únicamente para atender la solicitud del usuario, investigar problemas y mejorar el servicio.

El contenido enviado a funciones comunitarias puede utilizarse para mostrar, mantener, moderar y desarrollar el contenido comunitario.

7. Conservación de datos

Los datos almacenados localmente permanecen en el dispositivo del usuario hasta que los elimine en la aplicación, borre los datos de la aplicación o desinstale la aplicación del dispositivo.

Las solicitudes de soporte enviadas por correo electrónico se conservan el tiempo necesario para gestionar el asunto, prestar atención al cliente y responder posibles preguntas de seguimiento.

Las contribuciones comunitarias publicadas pueden conservarse como parte del contenido comunitario, salvo que el usuario solicite su eliminación o exista otro motivo para retirarlas.

8. Derechos del usuario

En virtud del RGPD, el usuario puede tener derecho a:

  • Acceder a sus datos personales
  • Solicitar la rectificación de datos inexactos
  • Solicitar la supresión de datos personales
  • Solicitar la limitación del tratamiento
  • Oponerse al tratamiento de datos personales
  • Recibir los datos en un formato portable cuando corresponda

Dado que los datos principales de la aplicación se almacenan por defecto en el propio dispositivo del usuario, este puede gestionar, eliminar y borrar la mayor parte de los datos directamente en la aplicación o mediante los ajustes del dispositivo.

Si el usuario ha publicado una contribución comunitaria y desea solicitar su eliminación, puede contactar con support@sparkmaster.app.

9. Seguridad

Los datos locales del usuario están protegidos por la seguridad del propio dispositivo. Recomendamos:

  • Utilizar bloqueo del dispositivo, como PIN, contraseña o bloqueo biométrico
  • Mantener actualizados el sistema operativo y la aplicación
  • Realizar copias de seguridad del dispositivo con regularidad
  • No compartir fotos o notas que contengan datos personales sensibles o innecesarios

10. Menores

Sparkmaster no está dirigida a menores. La aplicación está pensada para su uso en relación con el mantenimiento y servicio de motores fuera borda.

Sublime Web Oy no recopila a sabiendas datos personales de menores. Si cree que se nos han facilitado datos personales de un menor, por ejemplo mediante una solicitud de soporte o una contribución comunitaria, contacte con support@sparkmaster.app.

11. Transferencias internacionales

Los datos locales de la aplicación permanecen en el dispositivo del usuario. Si el usuario envía correo electrónico, utiliza pagos de Google Play o publica una contribución comunitaria, los datos pueden pasar por los sistemas de los proveedores correspondientes y procesarse fuera de la UE/EEE según los términos y prácticas de privacidad de esos proveedores.

12. Cambios en esta política

Esta Política de privacidad puede actualizarse a medida que evolucionen la aplicación y el sitio web. Los cambios importantes se indicarán en la aplicación o en el sitio web antes o cuando entren en vigor.
