1. アプリの目的

Sparkmaster は船外機のメンテナンス・サービス支援アプリです。複数のエンジンメーカーに対応し、ユーザーはエンジン情報、メンテナンス履歴、点検、写真、その他ボート関連のメモを記録できます。

Sparkmaster は主に オフライン優先（offline-first） アプリとして設計されています。つまり、アプリの中核データは既定ではユーザー自身の端末に保存され、ユーザーの別途の操作または同意なしに Sublime Web Oy のサーバーへ送信されません。

2. ウェブサイト

Sparkmaster.app は静的ウェブサイトです。ユーザーアカウント、フォーム、決済機能は提供しておらず、サイト自体が直接個人データを収集することはありません。

ユーザーがメールでお問い合わせした場合、Sublime Web Oy は送信者のメールアドレス、メッセージ内容、添付ファイルなど、ユーザーが送信することを選んだ情報を受け取ります。

3. アプリが処理する情報

3.1 端末に保存されるデータ

アプリは、ユーザーの端末に次の種類のデータを保存する場合があります：

  • エンジン情報（モデル、シリアル番号、年式、使用時間、ボート名など）
  • メンテナンス履歴（実施した作業、日付、部品、作業手順、個人メモなど）
  • ユーザーが追加する 写真（エンジン、部品、作業手順など）
  • チェックリストおよびサービスメモ
  • ユーザーが手動でトリップ記録を開始するか、任意のマリーナ自動追跡を有効にした場合の トリップデータ（GPS 座標、航海時間、エンジン時間、任意の目的地など）
  • ユーザーが保存を選んだトラブルシュート結果や AI-Mechanic スキャン要約などの 診断履歴
  • 設定で入力した場合の 任意のワークショップ／サービス連絡先（メールなど）
  • コミュニティまたは投稿機能を利用する場合の ランダム識別子（UUID）

これらのデータは既定ではユーザーの端末にローカル保存されます。

3.2 位置情報

Sparkmaster は、ユーザーがトリップ記録を利用するか、任意のマリーナ自動追跡を有効にした場合にのみ位置情報を処理します。

手動トリップ記録： ユーザー自身が航海を開始します。航海中、アプリはエンジン時間と航海時間を計算するため、間引いた GPS ポイントを保存します。航海中は継続的な通知（フォアグラウンドサービス）が表示されます。画面オフ時も位置情報が読み取られる場合がありますが、航海がアクティブな間のみです。ルートポイントは端末に保存されます。

マリーナ自動追跡（任意）： ユーザーは別途、ホームマリーナ周辺のジオフェンスを有効にできます（航海 → マリーナ自動追跡）。定義したエリアから離れると航海が自動開始され、戻ると終了する場合があります。これは Android のジオフェンシングを使用し、アプリが閉じている、または画面がオフのときも確実に動作するためにバックグラウンド位置情報（ACCESS_BACKGROUND_LOCATION）が必要になる場合があります。既定では無効です。

権限と選択： 位置情報の権限は、アプリ内の説明をユーザーが確認した後にのみ要求されます。バックグラウンド位置情報はマリーナ自動追跡のためだけに要求されます。ユーザーは権限を拒否したり、端末設定でいつでも取り消したりできます。

位置情報は既定では Sublime Web Oy のサーバーへ送信されません。トリップ記録およびマリーナ自動追跡を使用しない場合、アプリは位置情報を処理しません。

3.3 写真

ユーザーはエンジン、部品、メンテナンス手順などの写真をアプリに追加できます。既定では写真はユーザーの端末に保存されます。

後からコミュニティ機能で写真を共有したり、不具合報告と一緒に送信したりする場合、個人データ、登録番号、顔、位置情報など、共有したくない内容が含まれていないことをユーザーが確認する責任があります。

3.4 コミュニティおよび投稿機能

ユーザーが部品互換情報、工具の使用経験、メンテナンスのヒント、写真などのコミュニティ投稿を公開することを選んだ場合、アプリはユーザーが明示的に共有を選んだ内容を処理・公開する場合があります。

これには次が含まれる場合があります：

  • ランダム識別子（UUID）
  • ユーザーが共有したテキスト
  • ユーザーが共有した写真
  • 部品番号、モデル情報、工具情報、その他ユーザーが任意で提供した内容

コミュニティ投稿は任意です。アプリが自動的に何かを公開することはありません。

ランダム識別子（UUID）はユーザーの氏名やメールアドレスではありませんが、永続的な技術識別子である場合があります。公開内容と紐づけ可能な場合は、予防的に個人データとして扱います。

公開されたコミュニティ投稿は、Creative Commons Attribution 4.0 International（CC BY 4.0） ライセンスの下で提供される場合があります。これは、ライセンス条件に従って元の投稿がクレジット表示される限り、他者が公開内容を利用できることを意味します。

公開前に、共有内容に個人データや公開したくない情報が含まれていないことを確認してください。

3.5 AI-Mechanic と端末上の AI

ユーザーが AI-Mechanic を有効にすると、Sparkmaster は症状への回答、選択した写真、短いエンジン音声、RPM 測定値、最小化されたエンジンおよび整備データの要約を処理する場合があります。AI 推論は端末上でローカルに実行されます。この内容は Sublime Web Oy や AI 提供者には送信されません。

分析に使用した元の写真と音声は、既定で分析後に削除されます。AI-Mechanic チャットに添付した写真も端末上のみで処理され、ターン終了後またはセッション終了時に消去されます（アップロードしません）。ユーザーは正常時の基準値を保持するか、結果を端末上の診断履歴に保存することを別途選択できます。AI チャットのメッセージは現在のセッションのみ保持され、チャットを離れると消去されます。チャット内の模式的な 3D 位置モデルはアプリ独自の図解であり、OEM の分解図ではありません。ダウンロードした AI モデルはローカルに保存され、設定から削除できます。Engine Health Passport はユーザーの確認後にのみ作成され、既定ではシリアル番号、正確な位置、元のメディア、自由記述メモを除外します。

AI-Mechanic は意思決定支援であり、確実な診断ではありません。安全上重要な警告には AI に加えて固定のアプリ規則を使用します。整備および安全指示は、エンジンの公式文書で確認してください。

4. 当社サーバーで収集しない情報

既定では、Sparkmaster は Sublime Web Oy 自身のサーバーに次を保存しません：

  • ユーザーアカウントまたはパスワード
  • ユーザーの氏名
  • 当社サーバー上の継続的な位置履歴
  • 広告ネットワーク向けの広告識別子
  • ソーシャルメディアプロフィール
  • マーケティングまたは広告向けの行動分析
  • 完全なメンテナンス記録、写真、トリップルート（ユーザーがメールで任意送信しない限り）

ユーザーが入力した場合、アプリは端末に 任意のワークショップメール をローカル保存する場合があります。不具合報告フォームはアプリ内でユーザーのメールを求めたり保存したりしません。ユーザーが自身のメールアプリから送信した場合、返信のため受信者にアドレスが表示される場合があります。

ユーザーがメールで不具合報告またはサポート依頼を送信した場合、メッセージ内容と送信者のメールアドレスは Sublime Web Oy に表示されます。

5. 第三者サービス

Sparkmaster は、マーケティング目的の 第三者広告ネットワークや行動分析を使用しません。

アプリは次の第三者サービスを使用する場合があります：

Google Play（アプリ内課金）

プレミアム機能は Google Play 経由で購入できます。Google が決済と購入記録を処理します。Sublime Web Oy はカード番号や銀行口座情報を受け取ったり保存したりしません。

Google 決済ポリシー：https://policies.google.com/payments

Firebase Crashlytics（Google）

Firebase が設定されたリリースビルドでは、安定性向上のため クラッシュおよびエラー診断 を Google Firebase Crashlytics に送信する場合があります。例：

  • クラッシュスタックトレースとエラーメッセージ
  • アプリバージョンとビルド番号
  • 端末モデルと OS バージョン
  • 選択したエンジンブランドやファミリーなどの非個人的な技術コンテキスト（利用可能な場合）

Crashlytics は、記録からメンテナンスメモ、写真、トリップルート、シリアル番号を収集するために 使用されません。アプリ設定（設定 → 匿名の利用データを共有）でクラッシュレポートをオフにできます。デバッグビルドでは既定で Crashlytics 収集は無効です。

Firebase Analytics（Google）

Firebase が設定されたリリースビルドでは、機能の利用状況を把握しアプリを改善するため、集約されたプロダクト分析 を Google Firebase Analytics に送信する場合があります。これは広告やマーケティングプロファイルには 使用されません。

送信される可能性のある例：

  • 部品情報を任意で確認したり、コミュニティフローで燃費数値を共有したりする際のカスタムイベント名
  • エンジンブランド、モデルファミリー、部品カテゴリ、信頼レベル、燃料単位、共有を選んだ数値的燃費値などの非個人パラメータ
  • Crashlytics と同様のアプリバージョンおよび技術コンテキスト

Analytics は、氏名、メール、シリアル番号、メンテナンスメモ、写真、トリップルート、正確な位置情報を 収集しません。アプリ設定（設定 → 匿名の利用データを共有）で分析をオフにできます。デバッグビルドでは既定で Analytics 収集は無効です。

Firebase / Google プライバシー情報：https://firebase.google.com/support/privacy

その他のネットワーク利用

アプリはユーザーのブラウザでリンク（販売店ページなど）を開いたり、UI に必要なフォント等をダウンロードしたりする場合があります。これらのサービスは広告プロファイル構築には使用されません。

クラウド同期やユーザーアカウントなど追加サービスを導入する場合、導入前または導入時に本プライバシーポリシーを更新します。

6. データの利用方法

アプリに保存されたデータは、主にユーザー自身のエンジン、メンテナンス、ボート情報を管理するために使用されます。

Sublime Web Oy は、メールで受け取ったサポート依頼および不具合報告を、ユーザーの依頼対応、問題調査、サービス改善の目的でのみ使用します。

コミュニティ機能に送信された内容は、コミュニティコンテンツの表示、維持、モデレーション、開発に使用される場合があります。

7. データの保持

ローカル保存データは、ユーザーがアプリ内で削除するか、アプリデータを消去するか、端末からアプリをアンインストールするまで端末に残ります。

メールで送信されたサポート依頼は、対応、カスタマーサポート、フォローアップ質問への回答に必要な期間保持されます。

公開されたコミュニティ投稿は、ユーザーが削除を求めない限り、または削除の他の理由がない限り、コミュニティコンテンツの一部として保持される場合があります。

8. ユーザーの権利

GDPR に基づき、ユーザーは次の権利を有する場合があります：

  • 個人データへのアクセス
  • 不正確なデータの訂正請求
  • 個人データの削除請求
  • 処理の制限請求
  • 個人データの処理への異議
  • 該当する場合、ポータブル形式でのデータ受領

アプリの中核データは既定ではユーザー自身の端末に保存されるため、ユーザーはアプリ内または端末設定から大部分のデータを直接管理・削除・消去できます。

コミュニティ投稿を公開したユーザーが削除を希望する場合は、support@sparkmaster.app までご連絡ください。

9. セキュリティ

ユーザーのローカルデータは、ユーザー自身の端末のセキュリティによって保護されます。以下を推奨します：

  • PIN、パスワード、生体認証などの端末ロックを使用する
  • OS とアプリを最新に保つ
  • 端末を定期的にバックアップする
  • 機微または不要な個人データを含む写真やメモを共有しない

10. 子ども

Sparkmaster は子ども向けではありません。本アプリは船外機のメンテナンスおよびサービスに関連して使用されることを想定しています。

Sublime Web Oy は、故意に子どもの個人データを収集しません。サポート依頼やコミュニティ投稿などを通じて子どもの個人データが提供されたと思われる場合は、support@sparkmaster.app までご連絡ください。

11. 国際的な移転

アプリのローカルデータはユーザーの端末に残ります。ユーザーがメールを送信したり、Google Play 決済を利用したり、コミュニティ投稿を公開したりする場合、データは該当サービス提供者のシステムを経由し、提供者の条件およびプライバシー慣行に従って EU/EEA 外で処理される場合があります。

12. 本ポリシーの変更

本プライバシーポリシーは、アプリおよびウェブサイトの進化に伴い更新される場合があります。重要な変更は、効力発生前または発生時にアプリまたはウェブサイトで示されます。
